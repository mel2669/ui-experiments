## Implementation in this project

This stat-cards app maps the tokens above into code as follows. When Figma variables change, update **`src/styles/design-tokens.css`** first, then reconcile **`src/styles/theme.css`** if any Tailwind or shadcn aliases need to stay in sync.

### CSS layers

| File | Role |
|------|------|
| `src/styles/design-tokens.css` | Source-of-truth CSS custom properties (`--svc-*`) for colors, overlay, and the **Default Card Style** shadow (`--svc-shadow-card`, equivalent to `0 2px 2px` at `#00000026`). |
| `src/styles/theme.css` | Bridges tokens into the app: shadcn `:root` theme (background, primary, borders, etc.) and **`@theme inline`** entries so utilities like `bg-svc-brand-primary`, `text-svc-success`, `border-svc-info`, `shadow-svc-card`, and `bg-svc-track` resolve from the same primitives. |
| `src/styles/fonts.css` | Loads **Roboto** (matches typography tokens). |
| `src/styles/index.css` | Import order: `fonts.css` → **`design-tokens.css`** → `tailwind.css` → `theme.css`. |

### React layout

| File | Role |
|------|------|
| `src/app/App.tsx` | Page shell (`bg-background`, max width, spacing). Renders the interactive playground and the static showcase. |
| `src/app/StatCardShowcase.tsx` | Stat card pattern gallery: cards use **`shadow-svc-card`**, semantic and brand colors (`svc-*`), grid **`gap-6`** (24px gutter, aligned with grid notes), and typography sizes aligned to **Column header** (13px medium), **Card & Modal** (22px section titles), and **H1 Medium** (28px primary metrics). |
| `src/app/CombinedFeaturePlayground.tsx` | Configurable combined card previews; accent borders use **`border-svc-info`**, **`border-svc-lavender`**, and **`border-svc-success`** instead of generic palette colors. |

### Practical mapping notes

- **Page background** uses Neutrals **Black 5** via shadcn `--background` (see `:root` in `theme.css`).
- **Card surfaces** use `bg-card` (white) with **steel**-aligned borders where applicable (`border-border` → `--border` / steel base).
- **Primary actions and brand emphasis** use `--primary` / **`text-svc-brand-primary`** and **`bg-svc-brand-primary-light-max`** for icon wells.
- **Semantic UI** (trends, errors, success bars) uses **`text-svc-success`**, **`text-svc-error`**, **`bg-svc-success`**, **`bg-svc-info`**, **`bg-svc-lavender`**, **`text-svc-warning`** (e.g. star ratings per **Warning** base), consistent with the semantic color list in this document.
